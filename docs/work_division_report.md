## جزئیات بیشتر سهم هر بخش

### تحلیل نیازمندی‌ها (۲۰٪ از کل پروژه)
- بررسی صورت پروژه و استخراج ۲۲ نیازمندی عملکردی
- تعریف ۱۲ نیازمندی غیرعملکردی (امنیت، کارایی، قابلیت استفاده)
- نگارش ۳ سناریوی کاربرد کامل با جریان اصلی و جایگزین

### طراحی سیستم (۲۵٪ از کل پروژه)
- طراحی ۵ جدول پایگاه داده با روابط مشخص
- ترسیم ۴ دیاگرام UML کامل (Use Case، Class، Sequence، Activity)
- طراحی معماری سه‌لایه (Presentation → Application → Data)

### پیاده‌سازی (۴۵٪ از کل پروژه)
- بکند: ۱۲ مسیر (route) شامل احراز هویت، CRUD آزمون، شرکت در آزمون و نتایج
- پایگاه داده: ۵ مدل SQLAlchemy با روابط One-to-Many
- فرانت‌اند: ۱۳ تمپلیت Jinja2 + فایل CSS اختصاصی (~۸۰۰ خط) + JS تعاملی

### تست و یکپارچه‌سازی (۱۰٪ از کل پروژه)
- تست رندر تمام تمپلیت‌ها با داده‌های نمونه
- رفع باگ‌های منطقی (جلوگیری از ارسال مجدد آزمون، تایمر شخصی هر دانشجو)
- بررسی امنیتی (جلوگیری از دستکاری نمره با ارسال تکراری فرم)
