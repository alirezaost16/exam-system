# مستند تحلیل سیستم — سامانه آزمون آنلاین

> درس مهندسی نرم‌افزار | دانشگاه | ۱۴۰۵

---

## ۱. معرفی سیستم

سامانه آزمون آنلاین یک پلتفرم مبتنی بر وب است که امکان طراحی، برگزاری و ارزیابی آزمون‌های دانشگاهی را به‌صورت کاملاً دیجیتال فراهم می‌سازد. این سامانه برای سه نوع کاربر طراحی شده است: **مدیر سیستم**، **استاد** و **دانشجو**.

---

## ۲. نیازمندی‌های عملکردی (Functional Requirements)

### ۲.۱ مدیریت کاربران

| کد | نیازمندی | اولویت |
|----|----------|--------|
| FR-01 | سیستم باید امکان ثبت‌نام کاربران جدید با نقش‌های مختلف را داشته باشد | بالا |

| FR-03 | مدیر سیستم باید بتواند نقش کاربران را تغییر دهد | بالا |
| FR-04 | مدیر سیستم باید بتواند کاربران را حذف کند | متوسط |
| FR-05 | کاربران باید بتوانند اطلاعات شخصی خود را مشاهده کنند | متوسط |

| FR-08 | استاد باید بتواند سوالات تشریحی به آزمون اضافه کند | متوسط |
| FR-09 | استاد باید بتواند نمره هر سوال را به‌طور مجزا تعریف کند | بالا |
| FR-10 | استاد باید بتواند مدت زمان آزمون را به دقیقه مشخص کند | بالا |
| FR-11 | سیستم باید آزمون‌ها را در بازه زمانی مشخص‌شده فعال کند | بالا |

### ۲.۳ شرکت در آزمون

| کد | نیازمندی | اولویت |
|----|----------|--------|
| FR-12 | دانشجو باید بتواند لیست آزمون‌های فعال را مشاهده کند | بالا |

| FR-14 | سیستم باید تایمر شمارش معکوس را در حین آزمون نمایش دهد | بالا |
| FR-15 | سیستم باید پاسخ‌ها را به‌صورت خودکار ذخیره کند | بالا |
| FR-16 | سیستم باید پس از اتمام وقت، آزمون را به‌صورت خودکار ارسال کند | بالا |
| FR-17 | دانشجو نباید بتواند در یک آزمون بیش از یک بار شرکت کند | بالا |

### ۲.۴ ارزیابی و نتایج

## ۳. نیازمندی‌های غیرعملکردی (Non-Functional Requirements)

### ۳.۱ امنیت
- **NFR-01**: رمزهای عبور باید با الگوریتم SHA-256 هش شوند
- **NFR-02**: نشست کاربری باید پس از ۲۴ ساعت منقضی شود
- **NFR-03**: دسترسی به صفحات محدود باید بر اساس نقش کاربر کنترل شود
- **NFR-04**: کوکی‌های نشست باید با پرچم HttpOnly ذخیره شوند

### ۳.۲ کارایی
- **NFR-05**: صفحات باید در کمتر از ۲ ثانیه بارگذاری شوند
- **NFR-06**: سیستم باید حداقل ۱۰۰ کاربر همزمان را پشتیبانی کند
- **NFR-07**: عملیات پایگاه داده باید بهینه باشد (استفاده از index)

### ۳.۳ قابلیت استفاده
- **NFR-08**: رابط کاربری باید به زبان فارسی و راست به چپ باشد
- **NFR-09**: سیستم باید در مرورگرهای مدرن به‌درستی نمایش داده شود
- **NFR-10**: رابط کاربری باید واکنش‌گرا (Responsive) باشد

### ۳.۴ قابلیت نگهداری
- **NFR-11**: کد باید به‌صورت ماژولار و مستندسازی‌شده باشد
- **NFR-12**: پایگاه داده باید با استفاده از ORM مدیریت شود

## ۵. معماری سیستم

```
┌─────────────────────────────────────────────────────┐
│                    Client (Browser)                  │
│         HTML/CSS/JS + Jinja2 Templates               │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP
┌──────────────────────▼──────────────────────────────┐
│                FastAPI Application                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│  │  Auth    │  │  Exams   │  │    Results       │  │
│  │  Router  │  │  Router  │  │    Router        │  │
│  └──────────┘  └──────────┘  └──────────────────┘  │
│  ┌─────────────────────────────────────────────┐    │
│  │          SQLAlchemy ORM Layer               │    │
│  └──────────────────────┬──────────────────────┘    │
└─────────────────────────┼───────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────┐
│                  SQLite Database                     │
│  users │ user_sessions │ exams │ questions │ results │
└─────────────────────────────────────────────────────┘
```

## ۶. مدل داده

### جداول پایگاه داده

| جدول | توضیح |
|------|-------|
| `users` | اطلاعات کاربران (id, username, password_hash, full_name, email, role) |
| `user_sessions` | نشست‌های فعال (token, user_id, expires_at) |
| `exams` | آزمون‌ها (title, description, start_time, end_time, duration_minutes, creator_id) |
| `questions` | سوالات (exam_id, question_text, type, points, options, correct_answer) |
| `exam_results` | نتایج (exam_id, student_id, score, answers, submitted_at) |
